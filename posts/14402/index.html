使用 Docker 部署常用的开发环境

2022-12-09 03:32:16 ，月黑风高的夜晚，当前写这篇 md 的状态，真的是 md 了！！！

上一秒我还在快乐的给毕设加缓存，忽然之间好好的一个 redis 说没就没，呜呜呜，我毅然决然决定加入 Docker 阵营。

目前的很多开发环境都可以使用 Docker 去直接部署，欸，非常舒服（并不是我懒）。

然后为了避免以后找不到文档，记录下常用开发环境的部署。

Redis

  1. 首先拉取镜像

    docker pull redis:版本号

    其中版本号为：latest 时，为自动选择最后更新的版本，此时版本号可省略，docker run 时同理。

  2. 在 Linux 服务器里创建配置文件目录以及持久化目录

    mkdir -p /home/redis/[port]/data

    • /home/redis ：配置文件的容器卷目录
    • /home/redis/[port]/data ：持久化文件目录，加入 [port] 主要是为了区分该持久化文件属于哪一个 redis 服务

  3. 导入配置文件

    主要注意以下配置

  4. 启动 Reids 容器

    • --restart=always 表示总是自动重启
    • --log-opt max-size=100m --log-opt max-file=2 表示限定 log 文件的参数
    • -p 端口号:6379 表示把服务器的端口映射到 Redis 容器的 6379 端口
    • --name 自定义容器名字
    • -v 挂载容器卷
    • -d 表示后台启动
    • --appendonly yes 代表开启 redis 持久化

  5. 使用 docker ps -a 查看运行状态

Nginx

  1. 拉取镜像

    docker pull nginx:版本号

    参考 Redis 的说明

  2. 创建容器卷目录

    mkdir -p /home/nginx/ {conf,html,logs}

  3. 在文件夹 /home/nginx/conf 下新建 nginx.conf 文件

  4. 运行容器并把 nginx 容器的配置文件挂载到主机

Mongo

  1. 拉取镜像

    docker pull mongo:版本号

    参考 Redis 的说明

  2. 创建容器卷目录

    mkdir -p /home/mongo/db

  3. 启动容器

RabbitMQ

  1. 拉取镜像

    docker pull rabbitmq:management

  2. 启动容器

    docker run -d -p 5672:5672 -p 15672:15672 --name rabbitmq rabbitmq:management

MySQL (用于 Nacos)

  1. 拉取镜像

    docker pull mysql:版本

  2. 下载完成后创建在 /home/mysql/ 下新建两个对应文件夹
    mkdir -p /home/mysql/{data,conf}

  3. 创建完成后在 conf 目录中新建 my.conf 文件进行配置

  4. 启动容器

  5. 执行 sql 语句创建 Nacos 配置数据库

Nacos

  1. 拉取镜像

    docker pull nacos/nacos-server:v版本

  2. 创建挂载目录

    /home/nacos/{logs,init.d}

  3. 创建配置文件

    vim /home/nacos/init.d/custom.properties

    并编写配置文件

  4. 启动容器
